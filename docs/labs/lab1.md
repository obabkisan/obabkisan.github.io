## Создание и развертывание статического сайта на базе MkDocs с публикацией на GitHub Pages

## Цель работы
- Освоить процесс создания статического сайта с использованием генератора документации MkDocs.
- Научиться организовывать структуру документации проекта (портфолио лабораторных работ).
- Изучить базовые принципы работы с системой контроля версий Git и платформой GitHub.
- Развернуть статический сайт с использованием механизма GitHub Pages на домене вида `username.github.io`.
- Освоить базовую настройку темы оформления и конфигурационного файла `mkdocs.yml`.
---

## Задание

### Основная часть
1. Создать публичный GitHub репозиторий для размещения сайта-портфолио.
2. Настроить GitHub Pages для публикации из каталога `/docs` ветки `main`.
3. Клонировать репозиторий на локальный компьютер.
4. Создать и активировать виртуальное окружение Python.
5. Установить MkDocs в виртуальное окружение.
6. Настроить файл `.gitignore`.
7. Создать новый сайт командой `mkdocs new source`.
8. Перейти в каталог `source` и запустить локальный сервер.
9. Выполнить сборку сайта в каталог `/docs`.
10. Выполнить коммит и отправку в удалённый репозиторий.

### Самостоятельная часть
1. Выбрать и подключить тему оформления
2. Настроить файл `mkdocs.yml`: название сайта, тема, структура меню через параметр `nav`.
3. Создать структуру страниц: главная страница, об авторе, раздел "Лабораторные работы".
4. Обеспечить корректную навигацию по сайту.
5. Выполнить повторную сборку и публикацию сайта.

## Код и команды 
1. Создание репозитория `obabkisan.github.io`
2. Клонирование репозитория 
```
git clone https://github.com/obabkisan/obabkisan.github.io.git
cd obabkisan.github.io
```
3. Настройка виртуального окружения и установка MkDocs 
```
# Создание виртуального окружения
python -m venv env

# Активация (Mac/Linux)
source env/bin/activate

# Установка MkDocs и темы 
pip install mkdocs
pip install mkdocs-material

# Создание .gitignore
open .
touch .gitignore
```
4. Создание структуры сайта
```
# Создание нового проекта в папке site (source)
mkdocs new site

# Переход в папку site
cd site

# Запуск локального сервера для проверки
mkdocs serve
```
5. Настройка mkdocs.yml
```
site_name: Портфолио Прозоровой Полины
site_url: https://obabkisan.github.io/

theme:
  name: material
  palette:
    scheme: slate
    primary: black      
    accent: indigo       
  features:
    - navigation.tabs   
    - navigation.sections
    - navigation.expand
  font:
    text: Roboto
    code: Roboto Mono
       
nav:
  - Главная: index.md
  - Обо мне: about.md
  - Проекты: projects.md
  - Лабораторные работы:
      - Лабораторная 1: labs/lab1.md
      - Лабораторная 2: labs/lab2.md
  - Контакты: contact.md

markdown_extensions:
  - codehilite 
```
6. Сборка сайта 
```
# В папке site 
mkdocs build -d ../docs
```
7. Коммит и публикация
```
# Возвращение в корневую папку репозитория
cd ..

# Добавление всех файлов
git add .

# Коммит
git commit -m 

# Отправка на GitHub
git push 
```
